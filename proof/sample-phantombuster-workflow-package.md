# Sample Phantombuster Workflow Package

This example shows how Phantombuster or a similar tool can be positioned safely inside SC64.

Phantombuster is an optional signal intake, list movement, enrichment, or workflow packaging layer. It is not positioned as uncontrolled LinkedIn automation.

## Member-Approved Use Case

A member approves 100 target companies and asks SC64 to package a monitoring workflow.

## SC64 Package

- approved company list
- fields to enrich
- source status
- review-needed flag
- sensitive-company flag
- contact relevance score
- relationship path notes
- recommended next action
- member approval column

## Allowed Support

- move approved lists between tools
- structure action queues
- enrich approved records for review
- package signal monitoring outputs
- prepare contact exports
- create workflow handoff notes

## Boundaries

- no blind outbound
- no automatic messaging from the member's identity
- no unauthorized scraping claims
- no public profile action without approval
- no trusted-pool entry until reviewed

Core rule:

> SC64 recommends and packages. The member approves and owns execution.
