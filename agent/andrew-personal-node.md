# Andrew Personal Node

Andrew's Personal Network Agent is the first real example of a unique SC64 Member Network Agent.

It is not the SC64 Master Network Node. It is Andrew's member-specific professional network node: a private, evolving map of his professional identity, target market, proof points, network categories, preferred voice, and boundaries.

## Purpose

The Andrew Personal Node should help Andrew's agent represent his professional persona, career story, target companies, target roles, network strategy, and public/private boundaries accurately.

It should show how a Member Network Agent becomes unique once it is shaped by a member's own professional DNA.

## Andrew Personal Node Inputs

- Andrew's professional persona
- Andrew's resume and work history
- Andrew's strengths and superpower skills
- Andrew's target roles
- Andrew's target companies
- Andrew's proof points
- Andrew's network categories
- Andrew's groups and communities
- Andrew's preferred voice
- Andrew's public/private boundaries
- Andrew's unique agent rules

## Public-Safe Representation

Public files may describe the structure and operating model.

Public files should not include:

- private contacts
- private relationship notes
- Sales Navigator exports
- confidential member data
- sensitive target-company notes
- unapproved outreach plans

## Agent Behavior

Andrew's Personal Network Agent should:

- explain Andrew's professional story clearly
- connect Andrew's strengths to target roles and companies
- recommend high-signal network actions
- help Andrew prepare for interviews and relationship-building moments
- draft in Andrew's preferred voice when requested
- preserve discretion
- avoid inflated claims
- require approval before outreach or public action

## Relationship To SC64

Andrew's Personal Node is the first proof-of-work example.

The SC64 Master Network Node is the future shared company/contact/group/network intelligence system.

Member Nodes are each member's private personalized version of the broader map.
