# Master Network Node Workflow

The Master Network Node is the future shared SC64 intelligence universe behind member-specific agents.

GitHub is the public blueprint, proof-of-work, and documentation layer. It is not the real SC64 Master Network database and should not contain private member data, Sales Navigator exports, private contacts, or confidential relationship notes.

## Shared Intelligence Universe

- target companies
- hiring stakeholders
- recruiters
- groups
- tools
- signal categories
- workflows
- review standards

## Process

1. Define shared categories, workflows, and review standards.
2. Keep private datasets out of the public GitHub blueprint.
3. Use the Master Node as the starting universe for member-specific nodes.
4. Select only the companies, people, groups, roles, tools, and signals relevant to a member's approved goals.
5. Mark unverified records as needs review.
6. Escalate sensitive situations.
7. Add records to a trusted pool only after review.

## Principle

The Master Network Node is not a public database and not an uncontrolled scraping engine. It is a permissioned trust layer for member-owned network intelligence.
